# 催单功能配置说明

## 功能概述

催单功能允许买家在订单"待发货"状态时，向商家发送提醒消息，催促其尽快发货。

### 功能特性

1. **自动触发**：买家下单成功后，自动向商家发送一次提醒
2. **手动催单**：买家可在"待发货"状态下手动点击「催单」按钮
3. **8小时限制**：买家手动催单有8小时的频率限制
4. **消息模板**：使用微信订阅消息模板，包含订单号、商品、地址等信息
5. **跳转链接**：商家点击消息可直接跳转到订单详情页

## 催单限制说明

| 限制类型 | 规则 | 说明 |
|---------|------|------|
| 状态限制 | 仅限 `paid`（待发货）状态 | 其他状态无法催单 |
| 时间限制 | 8小时 | 手动催单后需等待8小时才能再次催单 |
| 自动触发 | 下单成功后立即触发 | 不计入8小时限制 |
| 商家接收 | 发送给所有授权商家 | `shop-settings.shop.sellerOpenIds` 列表中的所有商家 |

## 常见问题

### Q1: 为什么商家收不到消息？

**A:** 检查以下几点：
1. 商家的 OPENID 是否正确添加到 `shop-settings.shop.sellerOpenIds`
2. 模板 ID 是否正确配置
3. 商家是否在小程序中订阅了消息模板（首次需授权）
4. 云函数 `sendUrgencyMessage` 是否已部署

### Q2: 催单提示"太频繁"怎么办？

**A:** 系统限制8小时内只能催单一次，请等待冷却时间结束。

### Q3: 可以修改催单时间限制吗？

**A:** 可以。修改 `sendUrgencyMessage/index.js` 第 43 行：
```javascript
const eightHoursAgo = new Date(Date.now() - 8 * 60 * 60 * 1000);
// 改为其他时间，例如 24 小时：
// const oneDayAgo = new Date(Date.now() - 24 * 60 * 60 * 1000);
```

### Q4: 自动触发和手动催单有什么区别？

**A:** 
- 自动触发：下单后立即发送，不受8小时限制，不会记录 `lastUrgencyAt`
- 手动催单：需要买家点击按钮，受8小时限制，会记录 `lastUrgencyAt`

## API 调用示例

### 自动触发（下单后）
```javascript
wx.cloud.callFunction({
  name: 'sendUrgencyMessage',
  data: {
    orderId: 'order_id_here',
    autoTrigger: true  // 自动触发
  }
})
```

### 手动催单（买家点击）
```javascript
wx.cloud.callFunction({
  name: 'sendUrgencyMessage',
  data: {
    orderId: 'order_id_here',
    autoTrigger: false  // 手动触发
  }
})
```
